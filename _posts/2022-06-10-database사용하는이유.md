# Database 사용하는 이유

---
## FS(File system)
DBMS가 존재하기 전에는 File을 통해 데이터를 관리하였다. 

파일 시스템은 파일 관리를 용이하게 하기 위해 사용하는 시스템으로 여러 사람이 공유하고 사용할 목적으로 관리되는 데이터를 사용하기에는 적합하지 않은 점이 있다.

* 데이터의 중복 - 파일 시스템은 파일마다 데이터를 가지고 있어야 하므로 동일한 데이터를 서로 다른 파일에 저장한다.
* 데이터의 비일관성 - 데이터의 변경사항이 생기면 각 파일에서 해당 데이터를 모두 변경해야 하므로 수정 문제가 있고, 한꺼번에 수정되지 않으면 데이터간 inconsistency가 발생한다. 
* 데이터의 고립 - 정보가 여러 파일에 서로 다른 형식으로 흩어져 존재하면 재구성 비용이 들어가며 오랜 시간이 소요된다.
* 원자성 문제 - 프로그램이 수행중인 작업은 전부 완료되어 저장되어야 하며 완료되지 못하면 저장하지 말아야 한다.
* 동시성 문제 - 많은 사용자가 동시에 같은 정보에 접속해 정보를 수정한다면 정보의 일관성을 보장해야 한다.

---

## DBMS(Database management system)

이러한 문제점들을 해결하기 위해 DBMS는 다수의 사용자들이 데이터베이스의 데이터 생성, 접근, 관리유지할 수 있도록 전반적으로 지원하는 시스템이다. 이러한 파일시스템을 개선한 DBMS의 특징은 다음과 같다.
* Data의 독립성 - 데이터베이스의 크기를 늘리거나 성능 향상을 위해 데이터 파일을 늘리거나 새롭게 추가하더라도 관련 응용 프로그램을 수정할 필요가 없다.
* Data의 무결성 - 잘못된 Data가 발생하는 경우를 방지하는 기능으로 Data의 유효성 검사를 통해 Data 무결성을 구현한다.
* Data의 보안성 - 인가된 사용자만 DB의 자원에 접근할 수 있도록 계정 관리 및 접근 권한을 설정함으로써 모든 데이터에 보안을 설정할 수 있다.
* Data의 일관성 - 어떤 데이터가 수정되었을때 일부 Data만 수정되어 나머지 Data와 일치하지 않는 경우를 배제할 수 있다.
* Data의 중복 최소화 - 일관성과 마찬가지 맥락으로 데이터를 통합해서 관리함으로써 자료의 중복과 일관성 문제를 해결한다.
